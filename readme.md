<h2>Caso1: Tema e Problema de Pesquisa</h2>
Tema:
Implementação do padrão DAO em Java utilizando JDBC para conexão com banco de dados MySQL.

Problema de Pesquisa:
Como organizar e implementar uma camada de acesso a dados em Java utilizando o padrão DAO e JDBC de forma segura, estruturada e de fácil manutenção?<br>
<h2>Caso2: Relatório README.md</h2>
<br>
1. Padrão DAO (Data Access Object)O que é: É um padrão de projeto (design pattern) que isola a lógica de acesso a dados em uma camada separada.Para que serve: Serve para que as classes de "negócio" (como Cliente ou Pedido) não precisem saber como o banco funciona. Isso facilita a manutenção: se você mudar o nome de uma tabela, só precisa mexer no DAO, e não no sistema inteiro.Exemplo prático: Uma classe ProdutoDAO com o método buscarPorId(int id). O programador apenas chama esse método; o DAO cuida de abrir a conexão, rodar o SELECT e devolver um objeto pronto.2. Ciclo de Vida da Conexão JDBCAbrir conexão: Solicitar ao DriverManager o acesso ao banco usando a URL, usuário e senha.Criar statement: Preparar o objeto PreparedStatement, que conterá o comando SQL.Executar SQL: Enviar o comando ao banco (uso de executeQuery para buscar ou executeUpdate para alterar).Ler ResultSet: Caso seja uma busca, percorrer o objeto que guarda os resultados.Fechar recursos: Encerrar a conexão e os objetos de instrução para liberar memória do servidor.3. Práticas de SegurançaO que é SQL Injection: Uma vulnerabilidade onde um usuário mal-intencionado insere comandos SQL em campos de entrada para roubar ou apagar dados.Por que evitar concatenação: Somar strings ("SELECT * FROM user WHERE nome = '" + nome + "'" ) permite que o usuário escreva códigos no meio da sua consulta, alterando o comportamento do banco.Uso de PreparedStatement: É a solução definitiva. Ele usa parâmetros ? que tratam qualquer entrada como texto puro, impedindo que o banco execute comandos infiltrados.4. Checklist de QualidadeFechamento de conexão: Essencial para evitar vazamento de memória e travamento do banco por excesso de conexões abertas.Try-with-resources: Técnica do Java que declara os recursos (Connection, Statement) entre parênteses no try. O Java garante o fechamento automático ao final do bloco.Tratamento de exceções: Uso de blocos try-catch para capturar erros de banco (SQLException) e logar informações úteis em vez de derrubar a aplicação.Organização em camadas: Manter o código do banco no DAO e a lógica da aplicação em classes de Serviço (Service), garantindo que uma não interfira na outra.5.
